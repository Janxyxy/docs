---
title: "Normálové formy"
description: "Pravidla pro tvoření databází"
---

Tabulka nemusí nutně splňovat všechny normálové formy.

## 0. Normálová forma

- tabulka prostě existuje

## 1. Normálová forma

- Musí být v **normálové formě 0**
- Údaje v tabůlce jsou atomická, **dále nedělitelná**

:::tip Příklad

<table>
  <tr>
    <th>Titul</th>
    <th>Jméno</th>
    <th>Příjmení</th>
  </tr>
  <tr>
    <td>MuDr.</td>
    <td>Václav</td>
    <td>Ladislav</td>
  </tr>
  <tr>
    <td>null</td>
    <td>Štěpán</td>
    <td>Veliký</td>
  </tr>
</table>
:::

## 2. Normálová forma

- Musí být v **normálové formě 1**
- Všechny řádky by měli mít svůj vlastní **primární klíč**
- Všechny data musí být závislé na svém klíči
- Používá se ID od 0 do n

:::tip Příklad

<table>
  <tr>
    <th>ID</th>
    <th>Titul</th>
    <th>Jméno</th>
    <th>Příjmení</th>
  </tr>
  <tr>
    <td>0</td>
    <td>MuDr.</td>
    <td>Václav</td>
    <td>Ladislav</td>
  </tr>
  <tr>
    <td>1</td>
    <td>null</td>
    <td>Štěpán</td>
    <td>Veliký</td>
  </tr>
</table>
:::

## 3. Normálová forma

- Musí být v **normálové formě 2**
- Všechny **neklíčové** atributy jsou navzájem nezávislé

:::warning Příklad tabulky která není ve 3.NF

<table>
  <tr>
    <th>ID</th>
    <th>Jméno</th>
    <th>Příjmení</th>
    <th>Datum Narození</th>
    <th>Rodné číslo</th>
  </tr>
  <tr>
    <td>0</td>
    <td>Václav</td>
    <td>Ladislav</td>
    <td>25.1.2000</td>
    <td>000125/4040</td>
  </tr>
  <tr>
    <td>1</td>
    <td>Štěpán</td>
    <td>Veliký</td>
    <td>12.5.1997</td>
    <td>970512/7824</td>
  </tr>
</table>

### Z rodného čísla lze získat datum narození -> datum narození je závislé na rodném čísle -> nesplňuje 3.NF

:::
